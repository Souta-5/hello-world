# 2026ソフトウェア工学演習

## 第1回
授業の目標の説明。  
ソフトウェア工学の難しさ。

## 第2回
ソフトウェア工学の定義や基本的な考え方。  
ソフトウェアは単なるプログラムではなく、データ構造や使用方法を含めたもの。  
ソフトウェア工学は大規模化していて、社会では必要不可欠。

## 第3回
ソフトウェアライフサイクルについて。  
ソフトウェアは、ニーズの発生から要件定義、設計、開発、テスト、運用・保守、廃止までの流れで進む。  
ソフトウェア開発は複数人で行うことが多い。

## 第4回
プロジェクトの考え方と開発プロセスについて。  
プロジェクトは終わりがあり、独自の目標を達成する仕事。  
ウォーターフォール、スパイラル、反復型、アジャイルなどの開発手法があり、状況に応じて使い分けが求められる。

## 第5回
WBSについて。  
WBSはプロジェクトの作業を細かく分解し、誰が何をいつ行うのかを明確にするための手法。  
作業を洗い出すことで、やるべきことや担当範囲、スケジュールが整理され、プロジェクト全体を管理しやすくなる。

## 第6回
WBS課題の続き。  
実際に架空のイベントを実施するためのWBSを書いた。
